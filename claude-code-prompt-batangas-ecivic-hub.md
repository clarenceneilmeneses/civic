# Claude Code Prompt — Batangas City Youth E-Civic Hub

Copy everything below the line into Claude Code. Before running it, place the two seal files in the repo at `public/assets/` with the names referenced in the prompt (the brand icon itself will be generated by Claude Code as an SVG).

---

Build a full-stack e-civic hub website for **Batangas City, Philippines**, aimed primarily at helping **youth easily access activities and information about their local government**. The reference for structure and content types is the City Government of Naga website (naga.gov.ph) — it should feel like a typical Philippine city government website, but significantly easier to navigate, faster, and more modern. All public content must be manageable through a built-in CMS.

## Tech stack
- **Next.js 14+ (App Router, TypeScript)** with Tailwind CSS
- **Supabase** for Postgres database, Auth, Storage (file/image uploads), and Row Level Security
- Use `@supabase/ssr` for server-side auth. Generate typed DB helpers.
- Deployable to Vercel. Include a `.env.example` with `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`.
- Include SQL migration files (or a single `supabase/schema.sql`) I can run in the Supabase SQL editor, including RLS policies and seed data.

## Site structure (public)
Mirror the Naga City information architecture but simplified into clearer groupings:

**Navbar:** Home · Government (dropdown: Officials, Departments, SK/Youth Council) · Services · Youth Hub · Transparency · News · Contact — plus a persistent Search.

1. **Home**
   - Hero with city branding, tagline, and quick-action tiles (the #1 improvement over typical LGU sites): "Find an Event", "Get a Form", "Read Ordinances", "Emergency Hotlines".
   - "What's New in Batangas City" feed: latest announcements, bids & projects, executive issuances (CMS-driven).
   - Weather widget placeholder card (static mock data component, structured so an API can be plugged in later).
   - Featured city news cards with images.
   - Emergency hotlines strip (brick red section, matching Naga's "Emergency Hotlines" block).
   - Footer: seals/logos, GOVPH links, social links, UB research credit, subscribe form (stores emails in Supabase).

2. **Youth Hub** (the differentiator — make this shine)
   - Upcoming youth activities/events with category filters (Sports, Arts, Leadership, Volunteering, Scholarships, SK Programs).
   - Event detail pages with date, venue, organizer, and an **RSVP/registration button** (requires a free user account via Supabase Auth — email + Google OAuth).
   - Scholarship & opportunities board.
   - "Get Involved" section modeled on Naga's People Participation / i-Engage: proposed ordinances open for public comment; logged-in users can leave comments (moderated — hidden until approved).

3. **Transparency & Good Governance**
   - Ordinances, Resolutions, Executive Orders, Administrative Orders, Proclamations — filterable, searchable, paginated lists with PDF attachments from Supabase Storage.
   - Full Disclosure documents: Annual Budget, Bids & Projects, Financial Reports, Programs & Projects, Annual Investment Plans.

4. **Services**
   - Directory of city services and downloadable forms (PDFs in Storage), grouped by office, with plain-language "how to" steps per service.

5. **News** — article listing + article pages (cover image, rich text body, category, published date).

6. **Government** — city officials grid (photo, name, position), departments directory, SK Federation page.

7. **Contact** — office directory, emergency numbers page, contact form (submissions stored in Supabase, visible in admin).

## CMS / Admin (at `/admin`, protected)
- Supabase Auth with a `profiles` table and `role` enum: `admin`, `editor`, `citizen`.
- Admin dashboard with sidebar: Dashboard (counts/recent activity), News, Events, Legislation (ordinances/resolutions/issuances), Documents & Forms, Officials, Departments, Hotlines, Comments moderation queue, Contact submissions, Subscribers, Users (admin only).
- Full CRUD with a rich text editor (Tiptap) for article/event bodies, image upload to Storage with preview, draft/published status, and slug auto-generation.
- Editors can create/edit content; only admins can publish, delete, manage users, and moderate comments.
- RLS: public can `select` only published rows; citizens can insert their own RSVPs/comments and read their own; editors/admins per above. No service-role usage in client code.

## Database (suggested tables)
`profiles`, `posts` (news/announcements with `type` field), `events` + `event_registrations`, `legislation` (with `kind`: ordinance/resolution/executive_order/etc., number, title, summary, pdf_url, date), `documents` (transparency + forms, with category), `officials`, `departments`, `hotlines`, `proposals` + `proposal_comments`, `contact_messages`, `subscribers`. Add sensible indexes and `updated_at` triggers.
